Peta Materi

Apa yang Diujikan di UAS?

Seluruh soal Latihan UAS memakai satu data yang sama: [50, 93, 36, 72, 78, 100, 85, 18, 25, 38]. Yang diuji adalah kemampuanmu membangun struktur data dari array lalu memanipulasinya (hapus, sisip, pindah antar struktur) dan menelusuri hasilnya.

Strategi

5 Jebakan yang Paling Sering Muncul

  • Lupa mengarahkan ptr sebelum tambah() — pada kode dosen, sisip selalu terjadi setelah simpul yang ditunjuk ptr. Salah arah = salah posisi.
  • Menghapus head — head harus dipindahkan dulu ke simpul berikutnya; di DLL jangan lupa set head.prevNode = None.
  • Kondisi berhenti traversal — single/double: nextNode == None; circular: nextNode == head. Tertukar = infinite loop.
  • Urutan tampil stack vs urutan push — stack ditampilkan dari top, jadi urutannya terbalik dari urutan push. Queue tampil sesuai urutan masuk.
  • Delete by copying vs merging — copying: nilai suksesor disalin ke simpul (struktur hampir tidak berubah). Merging: anak kanan menggantikan simpul dan anak kiri "dicangkok" ke suksesor (struktur berubah besar).
Konsep

Single Linked List

Rantai simpul satu arah. Setiap simpul menyimpan dua hal: element (data) dan nextNode (alamat simpul berikutnya). Posisi pertama dipegang oleh head; simpul terakhir punya nextNode = None.

head →50●→93●→36●→...●→38●→None
  • Kosong jika head == None.
  • Hanya bisa ditelusuri maju — tidak ada jalan kembali ke simpul sebelumnya.
  • ptr adalah "jari penunjuk" — semua operasi sisip terjadi setelah simpul yang ditunjuk ptr.
Kode Kunci

Tiga Method Wajib Hafal

1. tambah(element) — sisip setelah ptr

def tambah(self, element):
    baru = self.simpulLL(element)
    if self.head == None:
        self.head = baru          # simpul pertama
    else:
        baru.nextNode = self.ptr.nextNode   # 1) baru menunjuk penerus ptr
        self.ptr.nextNode = baru            # 2) ptr menunjuk baru
Urutan tidak boleh dibalik! Jika ptr.nextNode = baru dijalankan dulu, alamat penerus lama hilang dan sisa rantai terputus.

2. telusuri(cari) — mencari simpul

def telusuri(self, cari):
    if self.head == None: return None
    tmp = self.head
    while tmp.element != cari:
        tmp = tmp.nextNode
        if tmp == None: return None   # sampai ujung, tidak ketemu
    return tmp

3. hapus(cari) — arahkan ptr ke simpul SEBELUM target

def hapus(self, cari):
    temp = self.telusuri(cari)        # target yang akan dihapus
    self.ptr = self.head
    while self.ptr.nextNode != temp:  # cari simpul sebelum target
        self.ptr = self.ptr.nextNode
    if self.head == temp:
        self.head = self.head.nextNode      # kasus hapus head
    self.ptr.nextNode = self.ptr.nextNode.nextNode  # lompati target
    del temp

Double Linked List

Setiap simpul punya dua pointer: nextNode (maju) dan prevNode (mundur). Akibatnya rantai bisa ditelusuri dua arah, dan menyisip sebelum sebuah simpul jadi mudah — tidak perlu mencari simpul sebelumnya secara manual.

head →None←●50●⇄●93●⇄●36●→...
Kode Kunci

tambah(element, maju=True/False)

Inilah pembeda terbesar dari single LL: parameter maju menentukan arah sisip terhadap ptr.

def tambah(self, element, maju=True):
    baru = self.simpulLL(element)
    if self.head == None:
        self.head = baru
    elif maju:                       # sisip SETELAH ptr (4 pointer diatur)
        baru.nextNode = self.ptr.nextNode
        baru.prevNode = self.ptr
        if self.ptr.nextNode != None:
            self.ptr.nextNode.prevNode = baru
        self.ptr.nextNode = baru
    else:                            # maju=False: sisip SEBELUM ptr
        baru.prevNode = self.ptr.prevNode
        baru.nextNode = self.ptr
        if self.ptr.prevNode != None:
            self.ptr.prevNode.nextNode = baru
        self.ptr.prevNode = baru
Hitung pointernya: sisip di tengah DLL selalu memperbarui 4 pointer — next & prev milik simpul baru, plus 1 pointer dari simpul kiri dan 1 dari simpul kanan.

hapus(cari) — manfaatkan prevNode

def hapus(self, cari):
    self.ptr = self.telusuri(cari)
    if self.ptr == self.head:                 # kasus head
        self.head = self.head.nextNode
        if self.head != None:
            self.head.prevNode = None
    else:
        if self.ptr.nextNode != None:         # sambungkan kanan ke kiri
            self.ptr.nextNode.prevNode = self.ptr.prevNode
        if self.ptr.prevNode != None:         # sambungkan kiri ke kanan
            self.ptr.prevNode.nextNode = self.ptr.nextNode
    del self.ptr

Circular Linked List

Sama seperti single LL, tetapi simpul terakhir tidak menunjuk None — ia kembali menunjuk head. Rantai menjadi lingkaran tanpa "ujung".

head →50●→93●→...●→38●→kembali ke head ↺
  • Simpul pertama menunjuk dirinya sendiri: saat list kosong, tambah() mengatur head.nextNode = head.
  • Traversal berhenti saat ptr.nextNode == head — bukan None!
  • Pencarian gagal terdeteksi saat tmp kembali ke head (sudah satu putaran penuh).

Perbedaan dengan Single LL — Hanya di 3 Titik

# 2) kondisi gagal pada telusuri
while tmp.element != cari:
    tmp = tmp.nextNode
    if tmp == self.head: return None   # satu putaran penuh

# 3) kondisi berhenti saat menampilkan
while not akhir:
    print(ptr.element)
    if ptr.nextNode == LL1.head:       # bukan == None
        akhir = True
    else:

Jebakan klasik: memakai kondisi nextNode == None pada circular LL menyebabkan infinite loop, karena tidak ada simpul yang pernah menunjuk None.

Stack — LIFO (Last In, First Out)

Seperti tumpukan piring: yang terakhir ditaruh adalah yang pertama diambil. Semua aktivitas terjadi di satu ujung saja, yaitu top.

OperasiArtiEfek pada top
push(el)menambah elemen di puncaktop = top + 1
pop()mengambil dan menghapus elemen puncaktop = top - 1
topEl()melihat elemen puncak tanpa menghapustetap
isEmpty()cek kosongtop == -1
Kode Kunci

Stack Berbasis Array

class recPool:                    # versi array
    def __init__(self, dat, maxSize, top):
        self.dat = dat
        for i in range(maxSize): self.dat.append(0)
        self.maxSize = maxSize
        self.top = -1             # -1 = kosong
    def isEmpty(self): return self.top == -1
    def topEl(self):   return self.dat[self.top]
    def pop(self):
        if self.top > -1:
            temp = self.dat[self.top]
            self.top -= 1
            return temp
    def push(self, el):
        if self.top + 1 < self.maxSize:   # cek penuh
            self.top += 1
            self.dat[self.top] = el

Versi linked list (LLStack) tidak punya batas penuh; push cukup: baru.nextNode = top; top = baru dan kosong dicek dengan top == None.

Queue — FIFO (First In, First Out)

Seperti antrean kasir: yang pertama datang dilayani pertama. Elemen masuk lewat belakang (last) dan keluar lewat depan (first).

Kode Kunci

Queue Array yang Sirkular — Kenapa?

Setiap deQueue menggeser first maju, meninggalkan slot kosong di depan array. Tanpa trik, queue akan "penuh palsu" padahal slot depannya kosong. Solusinya: saat first atau last melewati ujung array (maxSize - 1), pointer melingkar kembali ke indeks 0.

def deQueue(self):
    temp = self.dat[self.first]
    if self.first == self.last:        # elemen terakhir diambil
        self.first = -1; self.last = -1
    elif self.first == self.maxSize-1:
        self.first = 0                 # melingkar!
    else:
        self.first += 1
    return temp

def enQueue(self, el):
    if self.first == -1:               # queue kosong
        self.first = 0; self.last = 0
    elif self.last == self.maxSize-1:
        self.last = 0                  # melingkar!
    else:
        self.last += 1
    self.dat[self.last] = el

Binary Search Tree (BST)

Pohon biner dengan aturan tegas di setiap simpul: nilai lebih kecil → turun ke kiri; nilai lebih besar atau sama → turun ke kanan. Simpul pertama yang dimasukkan menjadi root.

def tambah(self, nilai):
    baru = self.simpul(nilai)
    if self.root == None:
        self.root = baru; return
    ptr = self.root
    while True:                       # turun sampai menemukan tempat kosong
        induk = ptr
        if nilai < ptr.nilai: ptr = ptr.left
        else:                 ptr = ptr.right
        if ptr == None: break
    if nilai < induk.nilai: induk.left = baru
    else:                   induk.right = baru
    baru.induk = induk

BST dari data UAS [50, 93, 36, 72, 78, 100, 85, 18, 25, 38]

            50
          /    \
        36      93
       /  \    /  \
     18   38  72   100
       \        \
        25       78
                   \
                    85

Coba telusuri sendiri: 85 masuk lewat jalur 50 → 93 (kiri, karena 85<93) → 72 (kanan) → 78 (kanan) → jadi anak kanan 78.

Pre-Order (root → kiri → kanan)

def preOrder(self, t):
    print(t.nilai)          # awal
    if t.left:  self.preOrder(t.left)
    if t.right: self.preOrder(t.right)

Root selalu muncul pertama. Berguna untuk menyalin/menyimpan struktur pohon.

Post-Order (kiri → kanan → root)

def postOrder(self, t):
    if t.left:  self.postOrder(t.left)
    if t.right: self.postOrder(t.right)
    print(t.nilai)          # akhir — root selalu muncul TERAKHIR
Trik hafalan: "in/pre/post" menunjuk posisi root dibaca — pre = root duluan, in = root di tengah, post = root paling akhir. Anak kiri selalu dikunjungi sebelum anak kanan di ketiganya.
Konsep Pendukung

Suksesor & Predesesor — Modal untuk Delete

  • Suksesor = simpul paling kiri dari anak kanan (cariMostLeftAnakKanan) — penerus terdekat yang lebih besar.
  • Predesesor = simpul paling kanan dari anak kiri (cariMostRightAnakKiri) — pendahulu terdekat yang lebih kecil.
  • Contoh pada BST UAS: suksesor dari root 50 = turun ke 93 lalu terus ke kiri: 93 → 72 → (72 tidak punya anak kiri) = 72.
Konsep

Menghapus Simpul BST: 3 Kasus Dasar

KasusCara menangani
Leaf (tanpa anak)putuskan dari induk: induk.left/right = None
1 anakanak naik menggantikan: induk langsung menunjuk anak
2 anakperlu strategi khusus → delete by copying atau delete by merging
Strategi 1

Delete by Copying — "Salin nilainya"

  • Cari suksesor (paling kiri dari anak kanan).
  • Salin nilai suksesor ke simpul yang dihapus — simpulnya tetap di tempat, hanya isinya berganti.
  • Hapus simpul suksesor yang asli (maksimal punya 1 anak kanan, mudah dilepas).
def deleteByCopying(self, simpul):
    suksesor = self.cariMostLeftAnakKanan(simpul.right)
    simpul.nilai = suksesor.nilai        # SALIN nilai
    if suksesor == simpul.right:         # suksesor anak kanan langsung
        simpul.right = suksesor.right
    else:                                # suksesor jauh di kiri bawah
        suksesor.induk.left = suksesor.right

Sifat: bentuk pohon hampir tidak berubah; aman dan sederhana, termasuk saat menghapus root (root-nya tetap, isinya yang baru).

Strategi 2

Delete by Merging — "Gabungkan subtree"

  • Anak kanan menjadi pewaris — menggantikan posisi simpul yang dihapus.
  • Seluruh anak kiri dicangkokkan menjadi anak kiri dari suksesor (simpul paling kiri di subtree kanan).
  • Jika yang dihapus root, pewaris menjadi root baru.
def deleteByMerging(self, simpul):
    pewaris = simpul.right               # anak kanan naik
    if simpul.left != None:
        suksesor = self.cariMostLeftAnakKanan(simpul.right)
        suksesor.left = simpul.left      # cangkok anak kiri
    if simpul == self.root:
        self.root = pewaris
    elif simpul.induk.right == simpul:
        simpul.induk.right = pewaris
    else:
        simpul.induk.left = pewaris

Sifat: bentuk pohon berubah signifikan (cenderung memanjang/tidak seimbang), tetapi tanpa menyalin nilai.

BST: Traversal & Dua Cara Delete

Kuadrat Sempurna — Bebas Pilih Struktur

"Habis diakar pangkat 2" = kuadrat sempurna: dari data, hanya 100 (10²), 36 (6²), 25 (5²). Sisanya bukan.

Strategi elegan (BST + stack):

  • Masukkan kuadrat sempurna ke BST pertama, sisanya ke BST kedua. Cek dengan int(x**0.5)**2 == x.
  • BST kedua: in-order langsung menghasilkan ascending.
  • BST pertama: in-order → push tiap nilai ke stack → pop semua = descending (stack membalik urutan).
